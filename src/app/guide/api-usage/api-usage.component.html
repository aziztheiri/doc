📡 API Usage Guide (Swagger)

This page helps you understand how to interact with our API directly through the Swagger interface.

1️⃣ Navigating the Swagger Interface

Go to the Swagger page to view all available endpoints grouped by functionality. Use the search bar to quickly find what you need.

2️⃣ Understand Each Endpoint

Click on any endpoint to expand it. You'll see:

  • HTTP Method: (GET, POST, PUT, DELETE)
  • URL: with any required path variables
  • Parameters: with their types, descriptions, and whether they are required

3️⃣ Entering Parameters

Before testing, you must enter required parameters. Tips:

  • 📌 Check if the parameter is in path, query, or body
  • 🧪 Body params are usually in JSON — click "Try it out" and edit the payload
  • 🔍 Required fields are marked with *

4️⃣ How to Edit or Submit New Data

For POST/PUT endpoints:

  • 📝 Click "Try it out"
  • 🧾 Modify the JSON payload in the request body
  • ⚠️ Respect data types (e.g. number, string, boolean)
💡 Tips:
  • Use the Authorize button to paste your JWT token first (if required)
  • If a request fails, check the error message and parameter types
  • Swagger is best for exploring and testing during development
