🧾 Bloc API - General Overview

🎯 Purpose & Use Cases

The Bloc API is primarily intended for administrative staff and backend services of the student housing system. It allows integration with other modules like room assignments, dormitory statistics, and maintenance tracking.

The Bloc API is designed to manage buildings in student housing. It provides full management operations such as: creation, modification, deletion, assigning to a dormitory, and associating rooms.

🌐 Base URL

All endpoints are accessible under the following base URL (example): 

https://api.insightweb.com/api

🔐 Authentication

All requests to the API require a valid JWT token to be included in the Authorization header: 

Authorization: Bearer <your-token>

📦 Main Endpoints

  • GET /bloc/all — List all blocs
  • POST /bloc/add — Add a new bloc
  • PUT /bloc/update — Update a bloc
  • DELETE /bloc/delete/{{'{'}}id{{'}'}} — Delete a bloc
  • POST /bloc/affecter-foyer/{{'{'}}idB{{'}'}}/{{'{'}}idF{{'}'}} — Assign a bloc to a dormitory
  • GET /bloc/find/{{'{'}}idB{{'}'}} — Retrieve a bloc by ID

🧬 Data Model - Bloc

A Bloc object consists of the following fields:

  • idB (Long) — Unique identifier (auto-generated)
  • nomB (String) — Bloc name
  • capaciteB (Long) — Bloc capacity
  • foyer (Object) — Associated dormitory (ManyToOne)
  • chambres (Array) — List of rooms (OneToMany)
